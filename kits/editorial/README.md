# Editorial

Editorial is a longform, type-led design system in the tradition of print
magazines and Sunday features. The default scheme is paper: near-white
background, near-black ink, and a single deep crimson accent that appears
only where it earns attention — a rule above a kicker, a drop initial, the
mark on a pull quote. Headlines are set in a serif display stack with tight
tracking and a large scale; body copy runs in a humanist sans with looser
leading for comfortable reading. Margins are generous, columns are narrow
enough to read in a single sweep, and whitespace does the structural work
that color and chrome do elsewhere. Reach for this system when the content
is the story — an essay, a feature, a manifesto, a long announcement — and
you want it to feel considered and printed rather than shipped.

## Composition

The chain reads the way a magazine feature unfolds top to bottom. **Masthead**
opens with the publication kicker, an oversized serif title, and the
byline/date line — the cover and dateline that establish voice before a word
of body copy. **ArticleHero** follows with the lead paragraph (set large, the
reader's entry into the piece) and an optional full-bleed lead image with a
caption, the first visual beat. From there the body alternates rhythm:
**ImageText** rows pair a media block with a block of prose and flip
left/right via a variant so consecutive rows never feel mechanical — the
media column usually holds a photograph, or a small bar/line chart when
the beat turns on numbers — while
**PullQuote** punctuates the read with an oversized centered line that lets
the eye rest and the argument land. **GalleryGrid** opens the page back up
into a multi-image spread — the visual exhale before the close. The piece
then turns outward: **Newsletter** offers the reader a way to keep reading
with an eyebrow, a headline, and an email capture, and **Footer** closes with
quiet section links and a colophon. Dense type-led beats and sparse
image-led beats trade off down the page, which is what gives a longform read
its pacing; scroll-triggered reveals carry each beat in as it enters so the
story unspools rather than arriving all at once.
